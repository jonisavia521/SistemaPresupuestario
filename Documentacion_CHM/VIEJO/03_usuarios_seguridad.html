Gestión de Usuarios y Seguridad

Administre usuarios, permisos y controles de acceso al sistema de forma segura.

Acceso al Módulo de Usuarios

Ruta: Menú Principal -> Archivos -> Usuarios

Solo los administradores tienen acceso a este módulo. Aquí puede crear, editar y eliminar usuarios del sistema.

Crear un Nuevo Usuario

Pasos para crear un usuario:

  1. Haga clic en el botón Nuevo
  2. Ingrese el nombre de usuario (sin espacios, máximo 50 caracteres)
  3. Asigne una contraseña temporal (el usuario deberá cambiarla en el primer acceso)
  4. Ingrese el nombre completo del usuario
  5. Seleccione el perfil/rol (Administrador, Vendedor, Aprobador, etc.)
  6. Asigne los permisos específicos si es necesario
  7. Haga clic en Guardar

Campos del Usuario

Campo Descripción Requerido
Usuario Identificador único del usuario en el sistema Sí
Contraseña Acceso seguro (mínimo 8 caracteres recomendado) Sí
Nombre Completo Nombre y apellido del usuario Sí
Email Correo para notificaciones No
Activo Habilita o deshabilita la cuenta Sí

Sistemas de Permisos

El sistema utiliza un modelo de permisos basado en **Familias y Patentes**:

Familia:

Agrupación de funcionalidades relacionadas (ej: Maestros, Cotizaciones, Facturación)

Patente:

Permiso específico dentro de una familia (ej: Ver Cotizaciones, Crear Cotizaciones, Aprobar Cotizaciones)

Roles Predefinidos

Rol Descripción Permisos Típicos
Administrador Acceso completo al sistema Todos los módulos, crear usuarios, configuración
Vendedor Crear y gestionar presupuestos Ver maestros, crear presupuestos, consultar precios
Aprobador Revisar y aprobar presupuestos Ver presupuestos, aprobar, rechazar, facturar
Consultor Acceso de solo lectura Ver presupuestos, ver facturas, consultar maestros
Jefe Ventas Gestión completa de ventas Todos los permisos de venta, reportes

Cambio de Contraseña

Procedimiento:

  • El sistema le pedirá cambiar la contraseña en el primer acceso
  • Debe ingresar una contraseña nueva (mínimo 8 caracteres recomendado)
  • Confirme la contraseña ingresándola dos veces
  • Haga clic en Aceptar
Seguridad:
  • Nunca comparta su contraseña con otros usuarios
  • Cambie su contraseña regularmente (cada 90 días recomendado)
  • Use contraseñas fuertes (letras, números, símbolos)
  • Si olvida su contraseña, contacte al administrador

Editar Usuario Existente

  1. En la lista de usuarios, seleccione el usuario a editar
  2. Haga clic en Modificar
  3. Actualice los campos necesarios
  4. Para cambiar contraseña, dejarla en blanco para mantener la actual
  5. Haga clic en Guardar

Deshabilitar Usuario

Para inhabilitar un usuario (sin eliminarlo):

  1. Seleccione el usuario
  2. Haga clic en Modificar
  3. Desmarque el campo Activo
  4. Haga clic en Guardar

El usuario no podrá iniciar sesión, pero sus datos históricos se mantienen en la base de datos.

Auditoría y Registros

El sistema registra automáticamente:

  • Quién creó o modificó cada registro
  • Fecha y hora de cambios importantes
  • Intentos de acceso fallidos (opcional)
  • Cambios en presupuestos y facturas
Consejo: Revise periódicamente los registros de auditoría para identificar actividades inusuales.

Mejores Prácticas de Seguridad

Recomendaciones:

  • Mínimo 3 administradores: Para evitar punto único de fallo
  • Permisos restringidos: Solo otorgue los permisos necesarios
  • Revisión periódica: Audite usuarios activos mensualmente
  • Eliminación de usuarios: Deshabilite cuentas no usadas
  • Cambios de rol: Actualice permisos si el usuario cambia de posición
  • Contraseñas fuertes: Exija cambio en primer acceso
  • Copias de seguridad: Realice backups regularmente

Solución de Problemas

Usuario olvidó su contraseña

Solo el administrador puede resetear la contraseña:

  1. Abra "Gestión de Usuarios"
  2. Busque y seleccione al usuario
  3. Haga clic en "Modificar"
  4. Ingrese una contraseña temporal
  5. Guarde los cambios
  6. Comunique la contraseña temporal al usuario de forma segura
  7. El usuario deberá cambiarla en el primer acceso

Usuario no puede acceder

Verifique:

  • ¿Está la cuenta Activa?
  • ¿Tiene el usuario los permisos necesarios para el módulo?
  • ¿Es correcto el nombre de usuario (sensible a mayúsculas)?
  • ¿Está la base de datos disponible?
